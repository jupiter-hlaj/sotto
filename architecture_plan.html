Call Routing

How the Right Cockpit Activates

Insurance agencies use a mix of direct lines and shared queues, and a mix of desk phones and softphones. The platform handles all scenarios.

Agent has their own dedicated phone number (DID). A 1:1 mapping in the admin portal connects this number to the agent's account. Simplest scenario.

📱 Client calls +1 (555) 234-0001
Sarah's direct line
→
☁️ Provider Routes to Sarah's
desk or softphone
→
⚙️ Webhook fires Payload includes
the called DID
→
🔍 Lookup DID → Agent ID
(admin mapping)
→
🧩 Sarah's Cockpit Activates instantly
ℹ️ Admin setup: In the portal, admin selects a number from their provider and assigns it to a specific agent. Done once. After that, all calls to that number auto-route to that agent's Cockpit regardless of whether they're on a desk phone or softphone.

Client calls a main number, hits an IVR, gets queued, and whoever picks up gets the call. We identify who answered from the provider's webhook payload, then activate that specific agent's Cockpit.

Main line

🤝 Agent answers Ext. 204 picks up

→
🔍 Lookup Ext 204 → Marcus
(admin mapping)
→
🧩 Marcus's Cockpit Only Marcus's activates
⚡ How Cockpit knows it's your call: The Chrome extension maintains a persistent WebSocket connection to the backend. When the backend resolves the answered_by to an agent_id, it pushes the call event only to that agent's WebSocket connection. Every other agent's Cockpit stays idle.

Most real agencies have both. Some agents have direct lines. Others are in a queue pool. Some use desk phones, others use softphones on their laptop or mobile. The admin maps each one.

Device Support
🖥️ Desk Phones (SIP) Identified by their SIP extension in the phone system. Admin maps extension → agent once during setup.
💻 Softphones Identified by their login email in the provider system. Matches directly to their agent account.
📱 Mobile Apps Provider mobile apps report the same user identity. Same routing logic applies.
✅ The backend doesn't care about device type. As long as the provider reports who answered (by extension, SIP URI, or user identity), the Cockpit routing works. Admin does the mapping once. Agents never touch configuration.

Ship fast, prove the loop, then add depth as clients pull you forward. Don't build Phase 3 before you have Phase 1 customers.

Prove the Loop

  • S3 storage per tenant
  • AWS Transcribe pipeline
  • Chrome side panel (Cockpit)
  • Transcript display
  • Notes input + save
  • Cognito auth + multi-tenancy

  • Admin number → agent mapping

Add Intelligence
